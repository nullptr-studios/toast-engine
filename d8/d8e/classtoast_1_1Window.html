Detailed Description

Singleton class managing the application window.

The Window class wraps GLFW to provide window creation, event handling, and OpenGL context management. It follows the singleton pattern as only one window is supported.

Features:
  • Window creation and destruction
  • Double-buffered rendering with buffer swap
  • OS event polling (keyboard, mouse, window events)
  • Framebuffer size queries
  • Display scaling for HiDPI support
Usage Example:
// Window is created automatically by Engine
auto* window = Window::GetInstance();
// Check framebuffer size
auto [width, height] = window->GetFramebufferSize();
// Check if minimized
if (window->IsMinimized()) {
// Skip rendering
}

This class is the main window class for the Toast Engine. It is a wrapper over the GLFW library.

To create a window simply call the constructor passing the window size and the name. There are a lot of features left to implement so if anything is needed that doesn't have a proper function expand the class wraping the function you need from GLFW. You should never use GLFW outside the HID module.
