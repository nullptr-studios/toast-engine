Detailed Description

A simple thread pool for executing jobs asynchronously.

The ThreadPool manages a fixed number of worker threads that process jobs from a queue. Jobs are executed in FIFO order, though completion order is not guaranteed due to parallel execution.

Usage Example:
ThreadPool pool;
pool.Init(4); // Create 4 worker threads
pool.QueueJob([]() {
// Do some heavy work...
});
// Wait for all jobs to complete
while (pool.busy()) {
std::this_thread::sleep_for(std::chrono::milliseconds(10));
}
pool.Destroy();

void toast::ThreadPool::Destroy ( )

Destroys the thread pool and waits for all workers to finish.

Signals all worker threads to stop and joins them. Any jobs still in the queue will NOT be executed.

Warning
This method blocks until all worker threads have terminated.
